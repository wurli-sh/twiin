# @twiin/shared — TypeScript Shared Library

Single source of truth for ABIs, deployed addresses, chain constants, ECDSA digest helper, and ERC-6551 address derivation. Consumed by `apps/backend` and `apps/frontend`.

**Status: Phase 2 complete — 22/22 vitest tests passing. Build compiles to `dist/`.**

## Commands

| Command           | Description                                                                                |
| ----------------- | ------------------------------------------------------------------------------------------ |
| `pnpm build`      | `tsc -p tsconfig.json` — compile to `dist/`                                                |
| `pnpm copy-abis`  | Extract ABIs from Hardhat artifacts → `abis/` (requires `pnpm compile` in contracts first) |
| `pnpm test`       | Run vitest parity tests                                                                    |
| `pnpm test:watch` | Vitest watch mode                                                                          |

## File Layout

```
packages/shared/
├── scripts/
│   └── copy-abis.ts              — reads artifacts/src/**/*.json, writes abis/*.json + abis/index.ts (as const)
├── abis/                         — generated by deploy.ts; do not edit by hand
│   ├── *.json                    — raw ABI arrays
│   └── index.ts                  — as const exports for full viem type inference
├── deployments/                  — generated by deploy.ts; deployment manifests per network
│   ├── hardhat.json
│   └── somniaTestnet.json
├── test/
│   └── parity.test.ts            — vitest tests (digest math, CREATE2, enums, sign→recover, Zod schema, manifest)
├── addresses.json                — auto-generated by contracts/scripts/deploy.ts; deployed Somnia testnet addresses
├── constants.ts                  — CHAIN_ID, TWIIN_6551_SALT, NativeConfigId, CapabilityId, enums, defaults
├── digest.ts                     — buildTwiinDigest()
├── twiin-account.ts              — deriveTwiinAccountAddress()
├── index.ts                      — barrel + loadAddresses(raw) + loadDeploymentManifest(raw)
├── tsconfig.json                 — CommonJS, ES2022, resolveJsonModule, outDir: dist
└── vitest.config.ts
```

## Key Exports

### `constants.ts`

- `CHAIN_ID = 50312`
- `TWIIN_6551_SALT` = viem `zeroHash` (bytes32(0))
- `NativeConfigId` — `{ JANICE:0, WEB_INTEL:1, ORACLE:2, ANALYSIS:3, REPORTER:4, EXECUTOR:5 }`
- `CapabilityId` — 9 entries computed at runtime via `keccak256(toBytes(name))`; matches `deploy.ts` exactly
- Plain enums mirroring `TwiinTypes.sol`: `TaskState`, `StepState`, `AgentLane`, `PlanMode`
- `DEFAULT_DAILY_CAP_WEI = parseEther("2")`, `DEFAULT_MAX_PER_TASK_WEI = parseEther("1")`, `DEFAULT_MAX_TRUSTLESS_WEI = parseEther("2")`

### `digest.ts` — `buildTwiinDigest(params): Hex`

Returns the **inner keccak256** (pre-EIP-191) matching `AgentOrchestrator.submitExternalResult`:

```
keccak256(encodePacked("\x19Twiin External Result v1\n", chainId, orchestrator, taskId, stepIdx, externalRequestId, keccak256(result)))
```

Sign with: `walletClient.signMessage({ account, message: { raw: buildTwiinDigest(p) } })`

### `twiin-account.ts` — `deriveTwiinAccountAddress(params): Address`

Deterministic ERC-6551 CREATE2 derivation:

- CREATE2 salt = raw `TWIIN_6551_SALT` (NOT a hash of the footer)
- Footer = `encodeAbiParameters([bytes32, uint256, address, uint256], [salt, chainId, twiinAgent, tokenId])` (abi.encode, 128 bytes)
- initCode = `concat([PROXY_HEADER(20), impl(20), PROXY_TRAILER(15), footer(128)])`

### `index.ts` — `loadAddresses(raw: unknown): Addresses`

Zod-validates and types the imported `addresses.json`. Fails loudly on stub. Pass imported JSON as argument — works in both Node and browser (no `require` at import time).

### `index.ts` — `loadDeploymentManifest(raw: unknown): DeploymentManifest`

Zod-validates full deployment manifests (addresses, txHashes, capabilities, nativeAgents, reservedNames).

## Exports Map

The `package.json` `exports` map enables direct imports:
- `@twiin/shared` — main barrel
- `@twiin/shared/addresses.json` — deployed addresses
- `@twiin/shared/deployments/hardhat.json` / `somniaTestnet.json` — manifests
- `@twiin/shared/abis/*.json` — individual ABIs

## Conventions

- viem `^2.47.4` for all EVM primitives — no ethers
- CommonJS module compiled to `dist/` via tsc; `files` includes `dist/`, `abis/`, `addresses.json`, `deployments/`
- `addresses.json`, `deployments/`, and `abis/` are written by `contracts/scripts/deploy.ts` — never edit by hand
- `privateKeyToAccount` is in `viem/accounts`, not `viem`
